系统概述

Luliy 博客是基于 Gmeek 框架的高度定制化个人博客系统。Gmeek 是一个利用 GitHub Issues 作为后端数据源、GitHub Pages 作为前端展示的轻量级静态博客生成器。

核心优势

  • 零数据库 — Issues 即数据源，纯静态生成，极速加载
  • 版本控制 — 所有内容用 Git 管理，天然支持历史追踪
  • 无服务器 — 直接部署到 GitHub Pages，无需维护服务器
  • 高度定制 — 通过三个配置文件完全控制外观和交互

技术栈

组件 技术
后端框架 Gmeek (GitHub Issues + GitHub Pages)
样式 CSS 3（Grid、Flexbox、自定义属性）

字体 LXGW WenKai（开源中文字体）
音效 Web Audio API
动画 CSS Transforms + JavaScript RAF

关键配置项：

{

  "themeMode": "manual",               // 主题模式

  "allHead": "<link>...<script>...",   // 全页面加载脚本

  "indexStyle": "<style>...",          // 首页内联样式
  "script": "<script>...",             // 文章页加载脚本
  "indexScript": "<script>..."         // 首页加载脚本

3. enhance.js — 脚本库（786 行）

全站交互逻辑引擎。无第三方框架依赖，原生 JavaScript 编写。

模块设计（14 个模块）：

① localStorage 初始化

自动保存用户设置：
├─ luliy-sfx: 音效开关 (default: '1' on)
└─ luliy-particles: 粒子背景开关 (default: '1' on)

② 顶部进度条 & 回顶按钮

window.addEventListener('scroll')
├─ 实时计算进度百分比
├─ bar.style.width = pct + '%'
└─ 300px 时触发按钮显示

③ 动态标题

document.addEventListener('visibilitychange')
├─ hidden 时：显示 "👀 别走呀，我还在进步！"
├─ visible 时：显示 "✨ 欢迎回来！ [原标题]" (2s 后复原)
└─ 用户体验提示

④ 运行时间

每秒更新一次：
├─ 计算距离 05/30/2026 的时长
├─ 显示格式：X天 X小时 X分 X秒
└─ 末尾秒数用红色高亮 (#ff4444)

⑤ 暗色模式切换波纹

监听主题按钮点击 (class "circle" 或包含 "theme")
├─ playSfx('theme') 播放三和弦音效
├─ 创建固定定位圆形元素
├─ 从中心以 2.2倍窗口对角线为半径扩展
├─ scale(0) → scale(1)，同时透明度 1 → 0
└─ 700ms 后自动移除

⑥ 粒子背景 Canvas

初始化 80 个粒子（彩虹色，hue 240-300）
每帧更新 (requestAnimationFrame):
├─ 鼠标悬停时粒子向鼠标吸引 (距离 < 220px)
├─ 鼠标靠近时粒子向外排斥 (距离 ≤ 60px)
├─ 粒子边界反弹
├─ 粒子之间相距 < 140px 时绘制连接线
├─ 鼠标位置绘制径向渐变光晕
└─ 暗色模式时透明度更高 (0.7 vs 0.45)

⑦ Web Audio 音效

四种音效（全部可配置音量和时长）：
├─ click: 下行方波 (900→400Hz, 60ms)
├─ sci: 正弦音扫描 (440→880→660Hz, 250ms)
└─ theme: 三和弦 (523/659/784Hz, 180ms每个，有延迟)

全局存储 _audioCtx 缓存，lazy 初始化
localStorage('luliy-sfx') === '0' 时禁用

⑧ 点击粒子火花

点击任意位置生成 14 个火花：
├─ 随机彩色小球（5种颜色）
├─ 从点击位置向外散射（随机角度、距离 18-73px）
├─ transition: 0.6s ease 消散并淡出
└─ 700ms 后自动清理 DOM

⑨ 头像时钟 & 链接到 About

检查流程：
1. 搜索 .avatar, img.avatar, .blogTitle img
2. 用 <a href="/about"> 包装头像
3. 在链接后插入 #luliy-avatar-clock div
4. 每 1000ms 更新一次 HH:MM:SS

最多尝试 20 次 (6 秒超时)，每次延迟 300ms

⑩ 标签页增强

仅在 /tag.html 页面激活：
1. 检测 #taglabel 容器
2. 在前面插入搜索工具栏
3. 实时过滤 .Label 元素
4. 显示匹配计数 (N / Total)
5. 监听 MutationObserver 自动重新计算

⑪ 图片灯箱

初始化固定遮罩层 #luliy-lightbox
事件流：
1. 代理监听 #postBody img 点击
2. 调用 open(src, alt) 激活遮罩层 + overflow: hidden
3. 点击遮罩层/×按钮/Esc 键时 close()
4. 300ms 后清空 src 防止内存泄漏

全局 API: window._luliyLightboxOpen(src, alt)

⑫ 文章页初始化 (_luliyInitPost)

使用 _luliyPostInited 布尔旗帜防止重复执行

⑫-a: 外链新标签 & 图片懒加载
  • <a href="http..."> 且不含博客域名 → target="_blank"
  • <img> → loading="lazy"

⑫-b: 阅读时长 & 字数（仅首次）
  • 创建 <p id="luliy-readmeta">
  • 字数 × 300 = 预计分钟数
  • 插入到 #postBody 最前面

⑫-c: 标题点击复制链接
  • h1, h2, h3 加 cursor: pointer
  • 点击：生成 URL = location + #heading_id
  • 复制到剪贴板 (navigator.clipboard)
  • 显示 "✓" 提示 2s

⑫-d: macOS 代码块控制栏
  • 三个 <button class="mac-btn">
  • 红(copy): playSfx + clipboard.writeText()
  • 黄(fold): pre.classList.toggle('is-folded')
    - 被折叠的代码块隐藏 code，底部 padding 减少
  • 绿(fullscreen): pre.classList.toggle('code-fullscreen')
    - fixed 全屏 (inset: 0)，z-index: 99999
    - 双击 pre 也触发（除了按钮）
    - Esc 键退出

⑫-e: TOC 折叠面板
  • 搜索 .toc, .markdown-toc, #markdown-toc
  • 创建 .luliy-toc-wrap 外壳
  • 在 wrap 前面插入 toggle 按钮
  • 移动整个 wrap 到 #postBody 最前面（readmeta 之后）
  • CSS 用 max-height 过渡：0 ↔ 600px
  • is-open class 切换展开状态

⑫-f: TOC 高亮 & 平滑滚动
  • scroll 事件检查当前可见的 h2/h3
  • 找到最后一个顶部位置 ≥ (scrollY - 130px) 的标题
  • 给对应 TOC 链接 <a> 加 .luliy-toc-active 类
  • 活跃状态样式：背景 rgba(240,180,41,0.22) + 文色 #d03a00
  • 点击 TOC 链接时：preventDefault + scrollTo({ behavior: 'smooth' })

⑫-g: 赞赏面板
  • <button> "✨ 和作者无限进步"
  • 渐变背景：yellow → pink
  • 点击展开 QR 码图片（max-height: 260px）

⑫-h: 上一篇 / 下一篇
  • 获取 postList.json，搜索当前文章的索引
  • 生成两个导航链接 (idx+1, idx-1)
  • 悬停效果：背景变深 + 上移 2px
  • 插入到 #postBody 末尾

⑬ 首页初始化 (_luliyInitIndex)

在首页或归档页激活：

a) 公告栏
  • 创建 <p class="luliy-announce">
  • 纯文本：只有一行"记录点滴 · 无限进步 🌱"
  • 无背景色，极简设计

b) 归档时间轴
  • 获取 postList.json 全部文章
  • 按年份分组
  • 时间轴样式：
    ├─ .tl-year: 大字标题（1.6rem，蓝色）
    ├─ .tl-list: <ul>，左边界线
    └─ .tl-item: <li>，左边黄色圆点标记

⑭ 主入口

ready() 时触发：
1. initLocalStorage()
2. initParticles()（如果未禁用）
3. 其他 7 个全局模块初始化
4. 检测页面类型：
   ├─ isPost: 调用 _luliyInitPost()
   ├─ isIndex/isArchive/hasList: 调用 _luliyInitIndex()
   └─ 都有 try-catch 保护

延迟执行 script 字段中的 _luliyInitPost()
（防止 DOM 未完全加载）

全站功能详解

🔍 站内搜索

触发方式：

  • 点击顶部右侧浮动按钮 ⌕（Search）
  • 按 / 快捷键（输入框外）

搜索流程：

1. 加载 postList.json
   ↓
2. 实时字符匹配（不区分大小写）
   • 搜索范围：标题 + 描述 + 标签 + 日期
   ↓
3. 显示最多 30 条结果
   • 匹配文章计数：N / Total
   ↓
4. 点击结果卡片跳转到文章页

UI 细节：

  • 搜索框上方显示统计 ("显示 N 篇" / "匹配到 N 篇")
  • 无结果时显示 "没有找到匹配内容"
  • 结果卡片包含：标题（蓝色）+ 描述（灰色）+ 标签（紫色小标签）

🎵 音效系统

四种音效：

名称 触发条件 声音特性
click 按钮/链接点击 快速下行方波 (900→400Hz)
sci 全屏代码、测试功能 正弦音扫描 (440→880→660Hz)
type 输入框输入 白噪声爆破 (20ms)
theme 主题切换 三和弦上升 (523/659/784Hz)

控制方式：

  • localStorage 存储开关状态 luliy-sfx: '0'/'1'

实现： Web Audio API，无外部音频文件

故障排查

问题 1：目录为空或不显示

原因： 文章中没有 H2/H3 标题，或 Gmeek TOC 插件未生成

解决方案：

# 这是 H1（不会出现在 TOC）

## 这是 H2 ← TOC 才认这个

### 这是 H3

问题 2：代码块样式不生效

原因： HTML 结构不符合预期（缺少 <pre><code> 嵌套）

排查步骤：

  1. 打开浏览器开发工具 (F12)
  2. 查看代码块 HTML 结构
  3. 确保 <pre><code class="language-xxx"> 格式

临时修复： 在 enhance.css 的代码块选择器后加 !important

问题 3：搜索功能找不到文章

原因： postList.json 未正确生成或文章未发布

检查清单：

  • Issue 已发布（不是草稿）
  • Actions 已运行完成（检查 Repository → Actions）
  • 打开 https://yourblog.github.io/postList.json 查看是否包含新文章

问题 4：音效不播放

原因： 浏览器禁用 Web Audio / 用户关闭音效开关

排查方法：

  1. 检查浏览器控制台是否有错误：console.log(getAudioCtx())
  2. 查看 localStorage：localStorage.getItem('luliy-sfx')
  3. 若为 '0'，点击音效图标长按 3 秒重新开启

问题 5：粒子背景卡顿

原因： Canvas 渲染性能不足（老电脑、移动设备）

解决方案：

// 在 localStorage 中禁用
localStorage.setItem('luliy-particles', '0')

然后刷新页面，粒子背景会被禁用，性能显著改善

问题 6：暗色模式样式错乱

原因： 浏览器缓存或 CSS 加载顺序

快速修复：

// 浏览器控制台运行
location.reload(true)  // 强制刷新（忽略缓存）

若仍未解决，检查 enhance.css 是否正确上传（网络标签查看）

问题 7：头像时钟不显示

原因： 头像元素不符合选择器，或 JS 未找到

排查方法：

  1. 浏览器开发工具搜索 .avatar 或 .blogTitle img
  2. 检查是否存在这些 class/结构
  3. 若不存在，修改 initAvatarClock 中的选择器
